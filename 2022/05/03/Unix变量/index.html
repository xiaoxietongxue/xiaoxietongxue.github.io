Unix变量

1.UNIX变量概述

变量是一种在你运行程序时将信息从外壳传递给程序的方式。程序在 “环境 “中寻找特定的变量，如果找到了这些变量，就会使用所存储的值。有些是由系统设置的，有些是由你设置的，还有一些是由shell或任何加载其他程序的程序设置的。

==标准UNIX变量被分成两类，环境变量和shell变量。==从广义上讲，shell变量只适用于shell的当前实例，用于设置短期工作条件；环境变量具有更深远的意义，那些在登录时设置的变量在会话期间有效。按照惯例，==环境变量的名字是大写字母，而shell变量的名字是小写字母。==

2.环境变量

环境变量的一个例子是OSTYPE变量。它的值是你当前使用的操作系统。键入

更多环境变量的例子有

  • USER (你的登录名)
  • HOME (你的主目录的路径名称)
  • HOST（你所使用的计算机的名称）
  • ARCH (计算机处理器的结构)
  • DISPLAY（用于显示X窗口的计算机屏幕的名称）
  • PRINTER（发送打印任务的默认打印机）
  • PATH (shell应该搜索的目录，以找到一个命令)

3.shell变量

shell变量的一个例子是历史变量。它的值保存的shell命令（保存的意义在于允许用户滚动查看他们以前输入的所有命令）。键入

更多关于shell变量的例子有

  • cwd (你当前的工作目录)
  • home (你的主目录的路径名称)
  • path (shell应该搜索的目录以找到一个命令)
  • prompt (用于提示交互式命令的文本字符串，shell是你的登录shell)

4.使用和设定环境变量

每次你登录到UNIX主机时，系统会在你的主目录中寻找初始化文件。这些文件中的信息被用来设置你的工作环境。C和TC shells使用两个文件，叫做.login和.cshrc（注意这两个文件名都是以点开头的）。

在登录时，C语言外壳首先读取.cshrc，然后是.login。

.login是用来设置适用于整个会话的条件，并执行仅在登录时相关的操作。

.cshrc用来设置条件，并执行shell和它的每个调用的特定动作。

准则是在.login文件中设置环境变量，在.cshrc文件中设置SHELL变量。
